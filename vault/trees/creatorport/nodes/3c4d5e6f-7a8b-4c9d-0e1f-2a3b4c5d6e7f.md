# Tech-Stack (Unity)

## Entscheidung

Unity als Engine. Entschieden am 2026-04-28.

## Begruendung

- Beste 3D-Performance auf Mobile (iOS + Android)
- WebGL-Build moeglich fuer Browser-Zugang
- Riesige Community und Asset-Ökosystem
- C# ist gut strukturiert fuer groessere Projekte
- STL-Export ueber Libraries moeglich
- AR Foundation fuer spaetere AR-Features
- Unity-Erfahrung vorhanden (Mittel-Level)

## Kern-Packages

| Package | Zweck |
|---|---|
| Unity UI Toolkit / uGUI | Touch-UI fuer Kids |
| Universal Render Pipeline (URP) | Performance auf Mobile |
| Addressables | Teile-Bibliothek nachladen |
| AR Foundation | Spaeter: AR-Vorschau |
| Unity Cloud Save | Designs speichern |

## STL-Export

Mesh-Daten aus Unity → STL-Writer (Custom oder Asset Store).
Kein externes Tool noetig, laeuft direkt in der App.

## Build-Targets

1. Android (Tablets, Priority)
2. iOS (iPads)
3. WebGL (Workshop-Fallback, kein Install)
