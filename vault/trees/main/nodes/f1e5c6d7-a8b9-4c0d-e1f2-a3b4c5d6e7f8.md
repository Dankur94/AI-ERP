# Cyber Resilience Act (CRA)

## Jedes Produkt mit digitalen Elementen betroffen

Von IoT bis Software — massive technische Anpassungen noetig.

## Timeline

| Datum | Ereignis |
|-------|----------|
| **11. Sep 2026** | **Pflichtmeldung von Cybersicherheitsschwachstellen** |
| **11. Dez 2027** | **Volle CRA-Compliance** |

## Betroffene

**Jedes Produkt mit digitalen Elementen**:
- IoT-Geraete (Smart Home, Industriesensoren)
- Software (SaaS, Desktop, Mobile Apps)
- Netzwerkgeraete (Router, Firewalls)
- Industriesteuerungen
- Medizingeraete (soweit nicht durch MDR abgedeckt)

## Pflichten

- **Security by Design**: Cybersicherheit ab Entwicklungsbeginn
- **Schwachstellen-Management**: Aktives Monitoring & Patching
- **Meldepflicht**: Schwachstellen innerhalb 24h an ENISA melden (ab Sep 2026)
- **Software Bill of Materials (SBOM)**: Vollstaendige Komponentenliste
- **CE-Kennzeichnung**: Cybersicherheits-Konformitaet nachweisen
- **Support-Zeitraum**: Mind. 5 Jahre Sicherheitsupdates

## Demand-Timing

- **Sep 2026**: Erste Meldepflicht — Unternehmen brauchen Incident-Response-Prozesse
- **2027**: Volle Compliance — umfassende Produktanpassungen
- **Laufend**: Schwachstellen-Monitoring als Dauerthema

## ERP-Relevanz

**Product Security Modul**:
- SBOM-Verwaltung (Software Bill of Materials)
- Schwachstellen-Tracking & Meldeworkflow
- Compliance-Checklisten pro Produkt
- CE-Konformitaets-Dokumentation
- Patch-Management-Uebersicht
