# Claude Vision API — Technische Details

## So funktioniert PDF-Extraktion mit Claude

Claude kann PDFs direkt verarbeiten (Vision-Faehigkeit). Zwei Wege:

### Weg 1: PDF als Base64 (empfohlen fuer Start)

```python
import anthropic
import base64

client = anthropic.Anthropic()

with open("rechnung.pdf", "rb") as f:
    pdf_data = base64.standard_b64encode(f.read()).decode("utf-8")

message = client.messages.create(
    model="claude-sonnet-4-6-20250514",
    max_tokens=4096,
    messages=[{
        "role": "user",
        "content": [
            {
                "type": "document",
                "source": {
                    "type": "base64",
                    "media_type": "application/pdf",
                    "data": pdf_data
                }
            },
            {
                "type": "text",
                "text": "Extrahiere alle Rechnungsdaten als JSON..."
            }
        ]
    }]
)
```

### Weg 2: Bild-Konvertierung (Fallback)

Falls PDF-Support Probleme macht:
1. `pdf2image` oder `poppler` konvertiert PDF → PNG
2. PNG als `image/png` an Claude senden
3. Funktioniert zuverlaessig auch bei gescannten Dokumenten

## Kosten-Kalkulation

| Modell | Input (1 Seite ~1500 Tokens) | Output (~500 Tokens) | Pro Rechnung |
|---|---|---|---|
| Claude Sonnet | $3/1M Input | $15/1M Output | ~$0.012 |
| Claude Haiku | $0.25/1M Input | $1.25/1M Output | ~$0.001 |

**100 Rechnungen/Monat mit Sonnet = ~$1.20**
→ Bei 49 EUR Verkaufspreis ist das ~2.5% COGS. Extrem gute Marge.

## Wichtig

- **Temperatur auf 0 setzen** — du willst deterministische Extraktion, keine Kreativitaet
- **Structured Output nutzen** — `response_format` oder JSON-Mode erzwingt valides JSON
- **Max 20 Seiten pro Request** — bei laengeren Dokumenten splitten
