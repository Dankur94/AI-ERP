# Odoo — Architektur-Lessons

| Odoo-Prinzip | Bedeutung fuer AI ERP |
|---|---|
| Schmaler Kern + Module | Kein Monolith, jede Loesung = 1 Modul mit Manifest |
| Direkte Modell-Erweiterung | Module erweitern sich gegenseitig, kein Event-Bus* |
| Bridge-Module | Optionale Integration, installiert sich auto wenn beide Module da |
| DB pro Kunde | Einfach, sicher, kein Daten-Leak |
| Community vor Enterprise | Erst Service-Geld, Lizenz ab Stufe 3 |

**Referenz:** TinyERP (2005) → OpenERP (2009) → Odoo (2014) → Enterprise Split (2015) → 500M EUR / 12M+ Nutzer (2025)

---

*\*Event-Bus = ein zentrales "Schwarzes Brett" im Code. Module posten Nachrichten dort, andere Module lesen sie — ohne sich direkt zu kennen. Odoo nutzt das NICHT. Stattdessen bauen sich Module direkt ineinander ein (z.B. Modul "Lager" erweitert direkt den Code von "Verkauf").*
