# Skills & SKILL.md

## Das Skill-System — OpenClaws cleverster Designentscheid

Skills sind der primaere Erweiterungsmechanismus. Man schreibt Instruktionen und Skripte — keine kompilierten Plugins, kein SDK noetig.

## Wie es funktioniert

### 1. Discovery
Beim Start scannt OpenClaw konfigurierte Skill-Verzeichnisse nach `SKILL.md`-Dateien.

### 2. Parsing
Jedes Skill-Frontmatter wird fuer Metadaten geparst:
```markdown
---
name: expense-report
description: Erstellt und verarbeitet Spesenabrechnungen
triggers:
  - Spesenabrechnung
  - Reisekosten
  - Auslagen
---

# Expense Report Skill

Du bist ein Experte fuer Spesenabrechnungen...
[Instruktionen fuer den Agenten]
```

### 3. Indexierung
Der vollstaendige Instruktionstexts wird fuer **Retrieval indiziert**.

### 4. Kontextuelle Aktivierung
Bei eingehender Nachricht entscheidet das System:
- **Explizit**: Nutzer aktiviert Skill direkt
- **Automatisch**: Matching basierend auf Nachrichteninhalt und Skill-Beschreibungen

### 5. Injection
Relevante Skill-Instruktionen werden in den Prompt-Kontext injiziert.

## Warum das genial ist
Es loest **Token-Effizienz** und **Modularitaet** gleichzeitig:
- Statt alle Tool-Beschreibungen im Prompt mitzuschleppen, wird nur das Noetige eingeblendet
- Neue Faehigkeiten hinzufuegen = neue Markdown-Datei erstellen
- Kein Rebuild, kein Redeploy

## Fuer AI ERP: Skill = Branchen-Template
- `SKILL.md` fuer "Handwerksbetrieb-Buchhaltung"
- `SKILL.md` fuer "E-Commerce-Fulfillment"
- `SKILL.md` fuer "Gastronomie-Warenwirtschaft"
- Kunden koennen eigene Skills schreiben — ersetzt SAPs komplexes Customizing
