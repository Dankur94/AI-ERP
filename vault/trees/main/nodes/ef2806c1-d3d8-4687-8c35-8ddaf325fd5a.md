# Dashboard-Idee — AI-generiertes Layout per Sprache/Text

**Kernidee:** Der Nutzer sieht ein leeres weisses Canvas. Er beschreibt per Text oder Spracheingabe, wie sein Dashboard aussehen soll. Die AI generiert es im Hintergrund.

---

## Konzept

### Ablauf
1. Nutzer oeffnet Dashboard zum ersten Mal → weisse Flaeche
2. Eingabefeld (Text) oder Mikrofon-Button (Voice)
3. Nutzer beschreibt: *"Zeig mir oben die offenen Rechnungen, darunter eine Liste der ueberfaelligen, rechts den Umsatz nach Monat"*
4. AI interpretiert die Beschreibung → generiert Widget-Layout
5. Dashboard erscheint — Nutzer kann weiter anpassen per Sprache/Text

### Interaktion
- **Text:** Chatfeld am unteren Rand oder als Overlay
- **Voice:** Mikrofon-Button, Speech-to-Text, dann gleicher Ablauf wie Text
- **Iterativ:** "Mach die Alerts groesser", "Tausch links und rechts", "Fueg einen CSV-Export-Button hinzu"

### Was die AI macht (Hintergrund)
- Uebersetzt natuerliche Sprache in Widget-Konfiguration (JSON)
- Waehlt aus verfuegbaren Widget-Typen (KPI-Card, Tabelle, Chart, Alert-Liste)
- Bestimmt Layout (Grid-Position, Groesse)
- Ruft bestehende API-Endpoints ab (GET /api/dashboard, /api/alerts, /api/reports/summary)

---

## Warum das passt

| Argument | Begruendung |
|---|---|
| **Differenzierung** | Kein Wettbewerber (Tofu, SAP Concur, Xero, Yonyou) hat AI-generierte Dashboards |
| **Persona-Problem geloest** | Jede Persona will andere Widgets zuerst sehen → statt Vorlagen lassen wir den Nutzer einfach sagen was er will |
| **L4-konform** | Progressive Disclosure — leeres Canvas ist maximal einfach, Komplexitaet kommt auf Nachfrage |
| **Kein neues Datenmodell** | Nutzt bestehende APIs, nur die Darstellung wird generiert |
| **WOW-Effekt** | Erste Nutzererfahrung ist beeindruckend statt "noch ein Standard-Dashboard" |

---

## Technische Skizze

### Widget-Typen (aus bestehenden Modulen)
- **KPI-Card:** Einzelne Zahl mit Label (Anzahl Rechnungen, Gesamtsumme, etc.)
- **Tabelle:** Rechnungsliste, Alert-Liste, Match-Ergebnisse
- **Status-Pipeline:** Hochgeladen → Extrahiert → Abgeglichen → Reported → Gewarnt
- **Chart:** Balken/Linie fuer Trends (Stufe 2 — erstmal ohne)

### AI-Prompt (vereinfacht)
```
Verfuegbare Widgets: [kpi_card, table, pipeline_status, alert_list]
Verfuegbare Datenquellen: [invoices, matches, reports, alerts]
Nutzer sagt: "{user_input}"
Generiere ein Dashboard-Layout als JSON.
```

### Output-Format (Beispiel)
```json
{
  "widgets": [
    {"type": "kpi_card", "source": "invoices", "metric": "count", "position": {"row": 0, "col": 0, "width": 1}},
    {"type": "kpi_card", "source": "invoices", "metric": "total_amount", "position": {"row": 0, "col": 1, "width": 1}},
    {"type": "alert_list", "source": "alerts", "position": {"row": 1, "col": 0, "width": 2}},
    {"type": "pipeline_status", "source": "invoices", "position": {"row": 2, "col": 0, "width": 2}}
  ]
}
```

### Voice-Input
- Web Speech API (Browser-nativ, kein externer Service)
- Alternativ: Whisper API (genauer, aber kostet)
- Speech-to-Text → dann gleicher Pfad wie Text-Input

## Abgrenzung zu bestehendem Dashboard-Plan

Der bisherige Plan (festes Layout mit 5-7 KPI-Cards) wird **nicht ersetzt**, sondern **ergaenzt**:
- **Standard-Dashboard:** Festes Layout als Default/Fallback (wie bisher geplant)
- **AI-Dashboard:** Optionaler Modus — "Customize with AI" Button
- Spaeter koennte AI-Modus der Default werden, wenn er gut genug ist
