# Modul 2 — Suche & Index

**Was:** Alles was ins System kommt wird indexiert und durchsuchbar. Volltextsuche ueber alle Dokumente.

**Warum als zweites:** Daten erfassen ohne sie wiederfinden zu koennen ist nutzlos.

## Scope

- [x] Dokumente in DB speichern
- [x] Volltextsuche ueber alle Felder (Lieferant, Rechnungsnummer, Dateiname)
- [x] Filter nach Datum, Lieferant, Status, Waehrung

## Status

Grundfunktion gebaut (2026-04-25). Backend + Frontend funktionsfaehig.

## Technische Details

- Backend: GET /api/invoices/search mit Query-Parametern
- Suche: SQLite LIKE (ausreichend fuer Stufe 1, FTS5 wenn noetig)
- Frontend: /search Route mit Live-Suche + Filter-Dropdowns
- L3: Alle Ergebnisse tenant-isoliert

## Spaeter

- Sortierung nach Relevanz
- Suche in Line Items
- Gespeicherte Suchfilter
