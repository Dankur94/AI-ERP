# Menschliche Faktoren

## Erlernbarkeit vs. Expertengeschwindigkeit

Zwei verschiedene Kurven. T-Codes: flache Expertenkurve (schnell, wenn man sie kennt), aber steile Anfaengerhuerde (auswendig lernen). Command Palettes brechen diesen klassischen Trade-off auf — der gleiche Mechanismus ist fuer Neulinge und Experten gut.

Wichtig ist auch die **Memorability** fuer Gelegenheitsnutzer: Jemand, der SAP alle drei Monate oeffnet, muss jedes Mal T-Codes nachschlagen. Ein Fuzzy-Launcher erfordert das nicht.

### Design-Implikation AI ERP
- Progressive Disclosure: Anfaenger und Experten nutzen denselben Mechanismus
- Keine Wissens-Voraussetzung fuer grundlegende Bedienung
- Muscle Memory bildet sich natuerlich ueber haeufige Nutzung

## Kognitive Last

Wie viel Zustand muss der Nutzer im Kopf halten?

| System-Typ | Kognitive Last | Warum |
|---|---|---|
| Modale Systeme (Vim) | Hoch | Nutzer muss Modus wissen |
| T-Codes (SAP) | Mittel | T-Code merken, Kontext kennen |
| Direct Manipulation | Niedrig | Zustand auf Bildschirm sichtbar |
| Command Palette | Niedrig | System zeigt Optionen |

Relevant vor allem bei Muedigkeit und unter Zeitdruck — da bricht modales Arbeiten zuerst. In ERP-Kontexten (Monatsabschluss, Jahresende) besonders relevant.

### Design-Implikation AI ERP
- Zustand immer sichtbar (Breadcrumbs, Status-Anzeige)
- Keine versteckten Modi
- Aktionen kontextabhaengig filtern, um Ueberflutung zu vermeiden

## Barrierefreiheit

| System-Typ | Accessibility |
|---|---|
| T-Codes (SAP) | Zufaellig gut (keine Maus noetig) |
| CLI-basiert | Gut (Screen-Reader-kompatibel) |
| Direct Manipulation (Figma) | Oft schlecht (Maus-abhaengig) |
| Command Palette | Sehr gut (Tastatur + Screen-Reader) |

### Design-Implikation AI ERP
- WCAG 2.1 AA als Minimum
- Jede Maus-Aktion per Tastatur erreichbar
- Screen-Reader-Kompatibilitaet von Anfang an
- Farbunabhaengige Statusanzeigen (nicht nur rot/gruen)
