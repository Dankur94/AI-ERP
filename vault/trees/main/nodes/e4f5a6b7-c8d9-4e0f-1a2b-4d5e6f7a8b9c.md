# Woche 4 — Login & verkaufbar

## Ziel

Das Tool hat Zugangsschutz und kann einem echten Kunden gezeigt (und verkauft) werden.

## Konkret

```
Browser:
  [Login]
  Email: david@importfirma.de
  [Magic Link senden]

  → Email: "Klick hier um dich einzuloggen"

  → Dashboard:
    Willkommen David!
    47 Rechnungen verarbeitet
    [Neue Rechnung hochladen]
    [Alle Rechnungen]
```

## Auth-Strategie: Magic Link statt Passwort

Warum kein Passwort-Login:
- **Weniger Code** — kein Passwort-Hashing, kein "Passwort vergessen"
- **Sicherer** — keine Passwoerter die geleakt werden koennen
- **Conversion** — kein Registrierungsformular, Email reicht

Implementierung: Token per Email → Cookie setzen → fertig.
Libraries: `next-auth`, `lucia-auth`, oder manuell (~50 Zeilen).

## Verkaufbar machen — Minimum

| Element | Warum | Aufwand |
|---|---|---|
| Landing Page (1 Seite) | Erklaert was das Tool tut | 2h |
| Demo-Video (60 Sek) | Zeigt den Workflow | 1h |
| Pricing (1 Plan) | "49 EUR/Monat, 100 Rechnungen" | 30min |
| Stripe Checkout | Bezahlung moeglich | 2h |

**Kein:** Freemium, Multiple Plans, Trial-Logik, Coupon-System.
Ein Plan. Ein Preis. Fertig.

## Deployment

Billigste Production-Umgebung:
- **Hetzner VPS** (4 EUR/Monat) oder **fly.io** (free tier)
- SQLite bleibt — kein DB-Server noetig
- HTTPS via Caddy oder fly.io automatisch
- Domain: ~10 EUR/Jahr

Gesamtkosten Production: **<15 EUR/Monat**
