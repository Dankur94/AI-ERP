# MoSCoW-Priorisierung

Must / Should / Could / Won't — fuer den MVP-Scope.

## Legende

| Kategorie | Bedeutung |
|---|---|
| **Must** | Ohne das ist das Produkt wertlos. Muss im MVP sein. |
| **Should** | Wichtig, aber MVP funktioniert auch ohne. Kommt direkt danach. |
| **Could** | Waere schoen, aber nur wenn Zeit uebrig ist. |
| **Won't** | Bewusst nicht. Nicht jetzt. Vielleicht spaeter. |

## Priorisierung

### MUST (MVP-Kern)
- [ ] PDF-Upload und automatische Datenextraktion
- [ ] Ergebnisse als strukturiertes JSON/CSV anzeigen
- [ ] Ergebnisse speichern und wiederfinden
- [ ] Login (Magic Link)
- [ ] Einfaches Dashboard

### SHOULD (direkt nach MVP)
- [ ] PO-Matching (Rechnung ↔ Bestellung)
- [ ] Multi-Dokument-Upload (Batch)
- [ ] Export-Funktionen (Excel, PDF-Report)
- [ ] Basis-Inventory-Tracking

### COULD (wenn Zeit)
- [ ] Influencer-Matching-Algorithmus
- [ ] Camera-Setup-Verwaltung
- [ ] Patientenakten-System
- [ ] WhatsApp-Integration
- [ ] Multi-Language-Support (EN/ZH)

### WON'T (bewusst nicht jetzt)
- [ ] Multi-Tenancy
- [ ] Rollen & Permissions
- [ ] Event Bus / DDD
- [ ] Mobile App
- [ ] Audit-Trail
- [ ] API fuer Drittanbieter

## Status

- [ ] MoSCoW-Priorisierung abgeschlossen
- [ ] MVP-Scope klar definiert und kommuniziert
