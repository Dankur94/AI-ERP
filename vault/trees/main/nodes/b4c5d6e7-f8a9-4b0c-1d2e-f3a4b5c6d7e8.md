# Journey Maps

Schritt-fuer-Schritt durch die Nutzererfahrung — vom ersten Kontakt bis zum taeglich genutzten Tool.

## Was ist eine Journey Map?

Visualisiert den Weg des Nutzers durch einen konkreten Workflow. Zeigt:
- **Schritte** — Was tut der Nutzer?
- **Touchpoints** — Wo interagiert er mit dem System?
- **Emotionen** — Wo ist er frustriert, wo zufrieden?
- **Pain Points** — Wo bricht der Prozess?
- **Opportunities** — Wo kann das Produkt eingreifen?

## Geplante Journey Maps

### Map 1: Invoice-Verarbeitung (David Wong)
```
Rechnung erhalten → Oeffnen → Daten ablesen → In PO suchen → Abgleichen → Kunden-Invoice erstellen → Zoll-Doku → Versenden
```
**Pain Points:** Manuelles Ablesen, Suchen in Ordnern, Copy-Paste-Fehler

### Map 2: Bestellung & Lager (Michelle Lau)
```
Kundenbestellung → Lagerbestand pruefen → Zusage/Absage → Kommissionieren → Lieferschein → Versand → Nachbestellen
```
**Pain Points:** Bestand unklar, Overselling, Excel-Chaos

### Map 3: Influencer-Besuch (Jenny Yeung)
```
Matching → Einladung → Influencer kommt → Camera-Setup → Content-Erstellung → Posting → Gaeste-Tracking → ROI-Auswertung
```
**Pain Points:** Kein Matching-System, kein ROI-Tracking, Content-Qualitaet

### Map 4: Patientenbesuch (Dr. Kevin Ho)
```
Patient kommt → Akte suchen → Vorgeschichte lesen → Behandlung → Dokumentation → Rezept → Abrechnung → Follow-Up
```
**Pain Points:** Akte nicht findbar, TCM+Western getrennt, manuelles Follow-Up
