# Validierung VOR dem Bauen

## Das Lean-Startup-Paradox

Du willst bauen. Aber die wichtigste Lean-Startup-Regel ist:
**Validiere bevor du eine Zeile Code schreibst.**

## Die 3 Fragen die beantwortet sein muessen

### 1. Existiert das Problem wirklich?

Nicht "Ich glaube Firmen haben das Problem" sondern:
- Hast du mit 5 Leuten gesprochen die Rechnungen manuell verarbeiten?
- Haben sie DIR gesagt dass es ein Problem ist? (Nicht du ihnen)
- Wie loesen sie es heute? (Excel? Praktikant? Ignorieren?)

### 2. Wuerden sie dafuer bezahlen?

Der haerteste Test. Nicht "Finden Sie das interessant?" sondern:
```
"Ich baue ein Tool das Ihre Rechnungen automatisch ausliest.
 49 EUR/Monat. Kann ich Sie auf die Warteliste setzen?"
```
Wenn 3 von 10 Ja sagen → bauen.
Wenn 0 von 10 Ja sagen → Pivot.

### 3. Kannst du sie erreichen?

Bestes Produkt bringt nichts wenn du keinen Zugang zur Zielgruppe hast:
- LinkedIn: Import/Export-Firmen, Buchhaltungsteams
- Branchengruppen: IHK, Import-Stammtische
- Steuerberater als Multiplikatoren

## Smoke Test (1 Tag Aufwand)

Bevor du Woche 1 startest:

1. **Landing Page** (Carrd.co, 30 Min, 0 EUR)
   - Headline: "Rechnungen automatisch auslesen. 3 Stunden/Woche sparen."
   - CTA: "Frueh-Zugang sichern" → Email sammeln
2. **5 LinkedIn-Nachrichten** an Import/Export-Leute
3. **1 Post** in relevanter Gruppe/Forum

**Ergebnis nach 48h:**
- 0 Signups → Problem existiert nicht (oder du erreichst die Falschen)
- 3+ Signups → Gruenes Licht fuer Woche 1

## Wenn du diesen Schritt ueberspringst

...dann baust du 4 Wochen lang etwas das vielleicht niemand will.
Das ist die teuerste Form von Selbstsabotage: produktiv aussehen, aber am Markt vorbei bauen.
