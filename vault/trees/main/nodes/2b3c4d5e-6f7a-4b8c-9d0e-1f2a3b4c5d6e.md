# Bedienparadigmen im Vergleich

## Ausgangspunkt: SAP T-Codes

Das T-Code-Modell optimiert eine Variable: **Tastaturanschlaege pro Aktion bei bekannter Aktion**. Dafuer ist es nach wie vor schwer zu schlagen — deswegen hat es in SAP bis heute ueberlebt.

Unterlegen ist es bei:
- Entdeckbarkeit
- Komposition
- Kontextbewusstsein
- Fehlertoleranz
- Unscharfe Absichten

## Uebersicht der Alternativen

| Paradigma | Staerke gegenueber T-Codes | Schwaeche |
|---|---|---|
| Command Palette | Entdeckbarkeit ohne Tempoverlust | Braucht gute semantische Labels |
| Unix-CLI mit Pipes | Komposition beliebiger Aktionen | Steile Lernkurve |
| Intent/NL (LLM) | Skaliert ueber unbekannte Aktionen | Nicht-deterministisch, langsamer |
| Kontextsensitiv | Reduziert kognitive Last | Modale Komplexitaet |
| Direct Manipulation | Sofortiges visuelles Feedback | Schlecht skalierbar, wenig accessible |

## Relevanz fuer AI ERP

Fuer das AI ERP ergibt sich ein Hybrid-Ansatz:
1. **Primaer:** Command Palette / Fuzzy-Launcher als Hauptinteraktion
2. **Power-User:** Tastenkuerzel / Shortcuts fuer Top-Aktionen
3. **Neu-Nutzer:** LLM-Intent-Interface als Einstiegshilfe
4. **Kontext:** System passt verfuegbare Aktionen an aktuellen Zustand an
