# Agent Core (Gehirn)

## Ebene 3 — Die Intelligenzschicht

Hier laeuft der eigentliche **Agent Loop**:

```
1. Nachricht empfangen (vom Gateway)
2. Kontext zusammenbauen:
   - Session-History (letzte Nachrichten)
   - Memory (persistiertes Wissen)
   - Relevante Skill-Instruktionen
3. LLM aufrufen (mit zusammengebautem Prompt)
4. Tool-Calls ausfuehren (falls LLM Tools aufruft)
5. Ergebnis zurueck an Gateway
6. Neuen Zustand speichern
```

## Kontext-Assembly
Der entscheidende Schritt ist das **Zusammenbauen des Kontexts**. OpenClaw optimiert hier auf Token-Effizienz:
- Nicht alle Skills werden geladen, nur relevante
- Aeltere Nachrichten werden komprimiert (siehe Memory-System)
- Tool-Ergebnisse werden selektiv eingeblendet

## Multi-Model-Faehigkeit
OpenClaw ist nicht auf ein LLM festgelegt. Der Agent Core abstrahiert den Model-Aufruf, sodass verschiedene Provider (OpenAI, Anthropic, lokale Modelle) austauschbar sind.

## Fuer AI ERP: Spezialisierte Agenten
Statt eines General-Purpose-Agenten braucht ein ERP **spezialisierte Agenten**:
- **Finance Agent**: Buchungen, Rechnungen, Mahnwesen
- **HR Agent**: Urlaubsantraege, Zeiterfassung, Onboarding
- **Procurement Agent**: Bestellungen, Lieferantenbewertung
- **Compliance Agent**: Regulatorik-Checks, Audit-Vorbereitung

Diese Agenten kommunizieren untereinander ueber den Gateway.
