# Die fuenf Hauptebenen

OpenClaw als Schichtenmodell mit klarer Trennung der Verantwortlichkeiten:

```
┌─────────────────────────────────────────────┐
│  1. Channel Layer (Eingang)                 │
│     WhatsApp, Telegram, Slack, Discord,     │
│     Signal, iMessage, Teams, Matrix, ...    │
│     50+ Kanaele mit eigenen Adaptern        │
├─────────────────────────────────────────────┤
│  2. Gateway (Hub)                           │
│     Sessions, Routing, Auth, Events         │
│     WebSocket-Server (Port 18789)           │
├─────────────────────────────────────────────┤
│  3. Agent Core (Gehirn)                     │
│     Agent Loop: Kontext → LLM → Tools → Out │
├─────────────────────────────────────────────┤
│  4. Tool/Skill Layer (Faehigkeiten)         │
│     Built-in + erweiterbar via Skills/MCP   │
├─────────────────────────────────────────────┤
│  5. Output Layer (Aktion)                   │
│     Host-direkt oder Docker-Container       │
└─────────────────────────────────────────────┘
```

## Designprinzip
Jede Schicht hat eine klare Aufgabe und kommuniziert nur mit ihren direkten Nachbarn. Das ermoeglicht:
- **Austauschbarkeit**: Neuer Kanal? Nur Adapter schreiben
- **Testbarkeit**: Jede Schicht isoliert testbar
- **Skalierbarkeit**: Schichten unabhaengig skalieren
