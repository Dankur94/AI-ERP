# Jobs to Be Done (JTBD)

Was will der Nutzer *wirklich* erreichen? Nicht Features, sondern Ergebnisse.

## Framework

> "When I [Situation], I want to [Motivation], so I can [Erwartetes Ergebnis]."

## JTBD pro Persona

### David Wong (Electronics Trading)
- When I **receive a supplier invoice**, I want to **automatically match it with my PO**, so I can **invoice my customer within minutes instead of hours**.
- When I **get a document request from customs**, I want to **find all related docs instantly**, so I can **avoid shipment delays**.

### Michelle Lau (Fashion/Textiles)
- When a **customer places an order**, I want to **see real-time stock levels**, so I can **never oversell again**.
- When **stock sits too long**, I want to **get alerted automatically**, so I can **discount it before it becomes dead stock**.

### Raymond Chan (F&B Import)
- When my **accountant asks for documents**, I want to **send everything sorted in one click**, so I can **stop paying for his search time**.
- When a **new EU regulation drops**, I want to **know immediately if I'm affected**, so I can **stay compliant without a lawyer**.

### Jenny Yeung (Restaurant)
- When I have **empty tables on Tuesday**, I want to **attract guests through targeted content**, so I can **cover my fixed costs every day of the week**.
- When an **influencer visits**, I want to **measure if their post actually brought guests**, so I can **invest in what works, not what looks good**.

### Dr. Kevin Ho (Healthcare)
- When a **patient walks in**, I want to **see their complete history in 3 seconds**, so I can **treat them instead of searching for files**.
- When a **PDPO audit is announced**, I want to **show compliance instantly**, so I can **focus on medicine, not paperwork**.
