Zwei zentrale Philosophien praeagen das AI-ERP-Projekt:

**Lean Startup** — Build-Measure-Learn. Schnell einen MVP bauen, echtes Feedback holen, iterieren. Kein Over-Engineering vor der Validierung.

**Domain-Driven Design (DDD)** — Die Software spiegelt die reale Geschaeftswelt wider. Die Fachsprache des Kunden wird 1:1 in Code uebersetzt. Module entstehen entlang fachlicher Grenzen, nicht entlang technischer Schichten.

Warum beide zusammen? Lean Startup sagt WAS zuerst gebaut wird (das Wichtigste fuer den Kunden). DDD sagt WIE es gebaut wird (so dass es spaeter skaliert, ohne alles umzuschreiben).
