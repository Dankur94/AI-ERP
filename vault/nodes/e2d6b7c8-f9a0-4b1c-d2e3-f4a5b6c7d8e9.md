# Gemeinsame Compliance-Logik

## Das wiederkehrende Muster aller EU-Regulierungen

Jede der 10+ neuen EU-Verordnungen erzeugt **denselben Typ von Beratungs- und Softwarebedarf**:

### 1. Datenpflicht
| Regulierung | Datenpflicht |
|-------------|-------------|
| AI Act | KI-System-Inventar, Risikoklassifizierung |
| EUDR | Geo-Koordinaten der Anbauflaechen |
| CRA | SBOM, Schwachstellen-Datenbank |
| E-Invoicing | Strukturierte Rechnungsdaten |
| DPP | Materialzusammensetzung, CO2-Fussabdruck |
| CBAM | Embodied Emissions pro Import |
| CSRD | Scope 1/2/3 Emissionen |
| DORA | ICT-Risiko-Register |

### 2. Drittverifikation
Akkreditierte Pruefer muessen die Daten validieren → Audit-Trail ist Pflicht in jedem Modul.

### 3. Default-Strafwerte
Wenn Unternehmen keine eigenen Daten liefern, gelten **hohe Default-Werte** (z.B. bei CBAM die hoechsten Emissionswerte des Herkunftslandes). → Starker Anreiz, eigene Daten zu erfassen.

### 4. Phased Rollout
Immer 2-5 Jahre: Erst Reporting-Pflicht (weich), dann Zahlungen/Strafen (hart). → Frueher Einstieg = Wettbewerbsvorteil fuer unsere Kunden.

### 5. Extraterritoriale Wirkung
Betrifft auch Nicht-EU-Unternehmen, die in den EU-Markt exportieren. → Globaler Markt fuer unsere Software.

## ERP-Architektur-Empfehlung

Baue eine **abstrakte Compliance-Engine** mit:
- Generischem Datenerfassungs-Framework
- Pluggable Verifikations-Workflows
- Konfigurierbarem Strafwert-Management
- Universeller Report-Engine (XBRL, PDF, API)
- Audit-Trail als Querschnittsfunktion
