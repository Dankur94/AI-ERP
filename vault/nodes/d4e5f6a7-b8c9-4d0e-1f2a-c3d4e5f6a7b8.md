# Gateway als Control Plane

## Funktion
Der Gateway ist das **zentrale Nervensystem** von OpenClaw — ein WebSocket-Server (Default-Port 18789), der alles orchestriert:

- **Session Management**: Haelt den Zustand aller aktiven Konversationen
- **Routing**: Leitet Nachrichten vom richtigen Kanal zur Agent Runtime
- **Authentifizierung**: Verifiziert eingehende Verbindungen
- **Event-Handling**: Verarbeitet Lifecycle-Events (Connect, Disconnect, Message, Error)

## Architektonische Entscheidung: WebSocket
WebSocket statt REST/HTTP bedeutet:
- **Bidirektionale Kommunikation** — Agent kann proaktiv Nachrichten senden
- **Persistent Connection** — kein Polling, niedriger Overhead
- **Real-time** — Antworten erscheinen sofort, nicht nach Request/Response-Zyklus

## Uebertragung auf AI ERP
Fuer ein ERP-Gateway muesste zusaetzlich gelten:
- **Multi-Tenancy**: Mandantenisolation auf Gateway-Ebene
- **Rollenbasiertes Routing**: Je nach Nutzerrolle andere Agenten/Skills aktivieren
- **Rate Limiting**: Schutz vor Ueberlastung bei vielen gleichzeitigen Nutzern
- **Event-Bus**: Nicht nur Nachrichten, sondern Geschaeftsereignisse routen
