# Agent Runtime

## Der Agent Loop
Die Agent Runtime ist das "Gehirn" von OpenClaw. Sie fuehrt die eigentliche KI-Schleife aus:

```
Nachricht rein
    ↓
Kontext laden (Session-History + Memory)
    ↓
LLM aufrufen (mit relevantem Kontext)
    ↓
Tool-Calls ausfuehren (gegen verfuegbare Capabilities)
    ↓
Neuen Zustand speichern
    ↓
Antwort zurueck an Gateway
```

## Capabilities (Built-in Tools)
- **Browser-Automatisierung**: Webseiten navigieren, Formulare ausfuellen
- **Dateisystem-Zugriff**: Lesen, Schreiben, Organisieren von Dateien
- **Shell-Ausfuehrung**: Beliebige Kommandos ausfuehren
- **Cron-Scheduling**: Zeitgesteuerte Aufgaben planen
- **Webhooks**: Externe Events empfangen und verarbeiten
- **Kamera & Screen-Recording**: Hardware-Zugriff (auf unterstuetzten Systemen)

## Isolation
Ausfuehrung der Aktionen geschieht entweder:
- **Direkt auf dem Host** (schnell, aber riskant)
- **In Docker-Containern** (isoliert, sicher — kritisch fuer Kundendaten)

## Fuer AI ERP
Die Agent Runtime muss fuer ERP um **spezialisierte Agenten** erweitert werden — nicht ein General-Purpose-Agent, sondern viele Domain-spezifische (Finance Agent, HR Agent, Procurement Agent), die untereinander kommunizieren.
