**Event Bus** — Ein zentraler Nachrichtenkanal. Komponenten senden Events ("es ist etwas passiert") und andere Komponenten hoeren zu und reagieren.

Warum? Entkopplung. Komponente A muss nicht wissen dass Komponente B existiert. A sagt nur: "Hey, ein Auftrag wurde erstellt." Wer auch immer sich dafuer interessiert, kann reagieren.

**Ohne Event Bus (schlecht):**
```
function erstelleAuftrag(daten) {
  auftrag = speichereInDB(daten);
  sendeEmailAnKunde(auftrag);        // hart verdrahtet
  aktualisiereBestand(auftrag);       // hart verdrahtet
  erstelleRechnung(auftrag);          // hart verdrahtet
  benachrichtigeLager(auftrag);       // hart verdrahtet
}
```
Problem: Jedes neue Feature bedeutet diese Funktion aendern. Alles haengt zusammen.

**Mit Event Bus (gut):**
```
function erstelleAuftrag(daten) {
  auftrag = speichereInDB(daten);
  eventBus.emit("AuftragErstellt", auftrag);  // fertig!
}

// Woanders, unabhaengig:
eventBus.on("AuftragErstellt", (auftrag) => sendeEmail(auftrag));
eventBus.on("AuftragErstellt", (auftrag) => aktualisiereBestand(auftrag));
eventBus.on("AuftragErstellt", (auftrag) => erstelleRechnung(auftrag));
```
Vorteil: Neues Feature? Einfach einen neuen Listener anhaengen. Nichts Bestehendes aendern.

**In eurer App ist ipcMain der Event Bus:**
- `ipcMain.on('tree:nodeSelected', ...)` — Tree sendet Event, main.js leitet an Editor weiter
- `treeWin.webContents.send('vault:refresh')` — main.js sendet Event an Tree
- `editorWin.webContents.send('marks:updated', ...)` — main.js informiert Editor ueber Aenderungen

**Domain Events** sind Events die fachliche Bedeutung haben (nicht technische):
- "AuftragErstellt" (Domain Event) vs. "buttonClicked" (UI Event)
- "RechnungBezahlt" (Domain Event) vs. "databaseUpdated" (technisches Event)

Im AI-ERP werden Domain Events der Klebstoff zwischen Modulen: Das Buchhaltungs-Modul reagiert auf "AuftragAbgeschlossen" vom Auftrags-Modul, ohne dass die Module sich kennen.
