# Modul 6 — Dashboard

**Was:** Startseite mit den wichtigsten Zahlen auf einen Blick. Konfigurierbar pro Branche.

**Warum zuletzt:** Zeigt Daten aus allen anderen Modulen. Ohne sie ist es leer.

## Scope

- [ ] KPI-Widgets (Pipeline-Status: hochgeladen → extrahiert → abgeglichen → reported → gewarnt)
- [ ] Daten aus Modul 1-5 aggregieren
- [ ] Alert-Zusammenfassung (ueberfaellige Rechnungen, Anzahl)
- [ ] Letzte Aktivitaet (zuletzt hochgeladene Rechnungen)
- [ ] Branchenspezifische Vorlagen — spaeter (Stufe 2)
- [ ] Konfigurierbare Widget-Reihenfolge — spaeter (Stufe 2)

## Status

Bereit zum Bauen. Modul 1-5 fertig. UX-Recherche dokumentiert (siehe Unterzweig).

## Design-Prinzipien (aus UX-Recherche)

- Max 5-7 KPI-Karten (Miller's Law)
- Wichtigste Zahl oben links (Fovea + F-Pattern)
- Farben fuer Status (rot/gelb/gruen) — Pre-attentive Processing
- Progressive Disclosure: Zusammenfassung zuerst, Detail bei Klick (passt zu L4)
- Grosse Klickziele (Fitts's Law), kein "tiny icon click"

## Technische Details

- Backend: GET /api/dashboard — aggregiert aus bestehenden Funktionen
- Kein neues Datenmodell — berechnet aus invoices, matches, reports, alerts
- Frontend: DashboardView.vue als neue Startseite (Route: /)
- Upload-Seite verschiebt sich auf /upload
- L3: Tenant-isoliert
