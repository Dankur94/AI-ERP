# Pricing & Packaging — Ein Plan, ein Preis

## Die einfachste Pricing-Strategie die funktioniert

```
+-----------------------------------+
|  Pro Plan — 49 EUR/Monat          |
|                                   |
|  ✓ 100 Rechnungen/Monat          |
|  ✓ Unbegrenzte Nutzer            |
|  ✓ JSON + CSV Export              |
|  ✓ Email Support                  |
|                                   |
|  [Jetzt starten]                  |
+-----------------------------------+
```

**Das ist alles.** Kein Free Tier, kein Enterprise Plan, keine jaehrliche Option.

## Warum 49 EUR?

| Faktor | Rechnung |
|---|---|
| Zeitersparnis Kunde | ~3h/Woche = 12h/Monat |
| Stundenlohn Sachbearbeiter | ~25 EUR/h |
| Wert fuer Kunden | ~300 EUR/Monat |
| Dein Preis | 49 EUR (~16% des Werts) |
| Deine Kosten (Claude API) | ~1.20 EUR/100 Rechnungen |
| **Marge** | **~97%** |

## Stripe-Integration (30 Minuten)

1. Stripe Account erstellen
2. Ein Produkt + Preis anlegen (49 EUR, monatlich)
3. Checkout-Link generieren
4. Webhook fuer `checkout.session.completed` → Nutzer freischalten

```python
# Checkout starten
session = stripe.checkout.Session.create(
    line_items=[{"price": "price_xxx", "quantity": 1}],
    mode="subscription",
    success_url="https://app.example.com/welcome",
    customer_email=user_email,
)
```

## Wann Pricing aendern?

- **>20 Kunden bei 49 EUR:** Experimentiere mit 79 EUR oder 99 EUR
- **Kunden fragen nach mehr:** Dann zweiten Plan einfuehren
- **Enterprise fragt an:** Individuelles Angebot, kein Self-Service

## Anti-Pattern vermeiden

- Kein "Kontaktieren Sie uns" — das ist eine Ausrede um kein Pricing zu setzen
- Kein Freemium — zieht die falschen Nutzer an und kostet API-Geld
- Keine komplizierte Feature-Matrix — verwirrt und verzoegert Kaufentscheidung
