# PDF-Fallstricke & Qualitaetssicherung

## Die zwei Welten von PDFs

### Digitale PDFs (Text ist Text)
- Von Software generiert (Word, SAP, Buchhaltung)
- Text ist direkt extrahierbar
- Claude liest diese perfekt
- **~70% aller Rechnungen**

### Gescannte PDFs (Text ist Bild)
- Papier → Scanner → PDF
- Kein selektierbarer Text
- Claude Vision erkennt trotzdem ~95% korrekt
- **Problem:** Schlechte Scans, schiefe Seiten, Stempel ueber Text

## Qualitaetssicherung — Vertrauen aber pruefen

### Stufe 1: Strukturelle Validierung (automatisch)
```python
def validate_invoice(data):
    errors = []
    # Pflichtfelder vorhanden?
    for field in ["lieferant", "rechnungsnummer", "brutto_gesamt"]:
        if data.get(field) is None:
            errors.append(f"Pflichtfeld fehlt: {field}")

    # Positionen summieren sich zum Gesamtbetrag?
    if data.get("positionen"):
        summe = sum(p["gesamtpreis"] for p in data["positionen"])
        if abs(summe - data.get("netto_gesamt", 0)) > 0.02:
            errors.append(f"Summe stimmt nicht: {summe} vs {data['netto_gesamt']}")

    return errors
```

### Stufe 2: Plausibilitaet (automatisch)
- Datum in der Zukunft? → Warnung
- Betrag negativ? → Warnung
- Waehrung passt nicht zum Land? → Warnung

### Stufe 3: Human Review (manuell)
- Zeige dem Nutzer PDF und extrahierte Daten nebeneinander
- "Stimmt das?" Button → bestaetigt oder korrigiert
- Korrekturen fliessen spaeter in Prompt-Verbesserungen ein

## Wichtig fuer MVP

**Nur Stufe 1 implementieren.** Stufe 2 und 3 kommen wenn Kunden danach fragen.
Lieber schnell raus mit 90% Genauigkeit als perfekt und nie fertig.
