# PDF-Extraktion & OCR — Grundlagen und Optionen

## Warum PDFs problematisch sind

Ein PDF ist **kein Dokument** — es ist eine **Zeichenanleitung fuer den Drucker**.

### Das Kernproblem

Adobe-Gruender John Warnock wollte 1993 ein Problem loesen: Jeder Drucker druckte anders. Seine Loesung: Hoer auf, dem Drucker zu sagen **was** der Inhalt ist. Sag ihm stattdessen **wie er es malen soll.**

```
Word-Dokument (strukturiert):     PDF (Zeichenanleitung):
<table>                            Zeichne "Kabel" bei (50, 200)
  <cell>Kabel</cell>               Zeichne "23.50" bei (300, 200)
  <cell>23.50</cell>               Zeichne Linie (40,195)→(400,195)
</table>
```

Was verloren geht:
- Dass "Kabel" und "23.50" zusammengehoeren
- Dass es eine Tabelle ist
- Dass 23.50 ein Preis ist
- Dass die Linie eine Tabellengrenze ist

**Ein PDF ist wie ein fertiges Gemaelde — du siehst das Ergebnis, aber nicht das Rezept. Und du kannst einen Kuchen nicht zurueck in Eier und Mehl zerlegen.**

### 3 Arten von PDFs

| Art | Inhalt | Schwierigkeit |
|---|---|---|
| Digital erstellt | Text als Zeichen-Befehle (Word→PDF) | Mittel |
| Gescannt | Nur ein Foto, null Text | Schwer — braucht OCR |
| Hybrid | Mix aus beidem | Am schwersten |

## Was ist OCR?

**OCR = Optical Character Recognition** — der Computer schaut auf ein Bild und erkennt Zeichen.

Warum OCR schwer ist:
- Schlechte Scans (schief, dunkel, Flecken)
- Handschrift vs. Druckschrift
- Aehnliche Zeichen: `l` vs `1` vs `I`, `O` vs `0`
- Chinesisch: ~50.000 Zeichen statt 26 Buchstaben
- HK-typisch: Chinesisch + Englisch gemischt auf einem Beleg

## Warum regelbasierter Code nicht reicht

Man koennte Regeln schreiben ("gleiche Y-Koordinate = gleiche Zeile"), aber:
- 0.3 Pixel versetzt — gleiche Zeile oder nicht?
- Keine Linien — trotzdem Tabelle?
- Jede Firma hat ein anderes Layout
- 1.000 Formate = 1.000 Regelsets, bricht bei Format 1.001

**AI-Vorteil:** Vision-Modelle schauen auf die Seite wie ein Mensch und rekonstruieren die Bedeutung per Mustererkennung — nicht per Koordinaten-Regeln.

### Empfohlene Zwei-Stufen-Pipeline

```
PDF → Mistral OCR 3 ($1-2/1k Seiten) → strukturierter Text → Claude Sonnet (Interpretation)
```

**Warum:**
- Mistral OCR macht die schwere OCR-Arbeit (besonders Chinesisch)
- Claude bekommt sauberen Text statt rohes PDF — billiger und genauer
- Kein Vendor Lock-in (kein Google/Azure noetig)
- Passt zu Stufe 1: einfach, modular, guenstig

**Alternative:** Claude direkt mit PDF bei niedrigem Volumen (<100 Seiten/Tag).
