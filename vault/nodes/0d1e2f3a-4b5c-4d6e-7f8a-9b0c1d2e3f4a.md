# Agenten-/Workflow-basiert

## Prinzip

Eine Absicht loest eine Kette von Teilschritten aus, moeglicherweise mit menschlichen Entscheidungspunkten. Gut fuer lange, unterbrechbare Prozesse — das ist das, wofuer T-Codes eigentlich nie gedacht waren.

## Beispiele

- **BPMN:** Business Process Model and Notation — visuell modellierte Workflows
- **Temporal:** Durable Execution — Workflow-Schritte ueberleben Systemneustarts
- **n8n / Zapier:** Low-Code-Workflow-Automatisierung
- **LangGraph / CrewAI:** AI-Agenten-Orchestrierung

## SAP-Kontext

SAP Workflow existiert, ist aber rudimentaer im Vergleich zu modernen Workflow-Engines:
- Schwer zu konfigurieren
- Eng an SAP-Objekte gebunden
- Debugging ist ein Alptraum
- Kein "Parken und spaeter weitermachen" fuer beliebige Prozesse

SAP-T-Codes sind fuer **einzelne, atomare Transaktionen** gedacht. Ein Beschaffungsprozess (Anforderung → Genehmigung → Bestellung → Wareneingang → Rechnung) muss muehsam ueber mehrere T-Codes und Workarounds (Parken, Vorerfassen) abgebildet werden.

## Vorteil

- **Langlebige Prozesse:** Tage, Wochen — der Workflow wartet
- **Menschliche Entscheidungspunkte:** Genehmigungen, Reviews, Freigaben
- **Transparenz:** Jederzeit sichtbar, wo ein Prozess steht
- **Handoff:** Halb fertiger Vorgang an Kollegen weitergeben — nativ moeglich

## Relevanz fuer AI ERP

- **Workflow-Engine als Kernkomponente:** Nicht nachgeruestet, sondern von Anfang an
- **AI-Agenten als Workflow-Teilnehmer:** LLM prueft Rechnung, Mensch gibt frei
- **Visueller Workflow-Editor:** Geschaeftsprozesse modellieren ohne Code
- **Statusverfolgung:** Dashboard zeigt alle laufenden Prozesse + Engpaesse
