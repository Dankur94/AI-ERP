# DB-Wahl — SQLite jetzt, Postgres spaeter

## Warum SQLite fuer MVP perfekt ist

| Eigenschaft | SQLite | Postgres |
|---|---|---|
| Setup | 0 — ist eine Datei | Server installieren, konfigurieren |
| Backup | Datei kopieren | pg_dump |
| Deployment | Mitliefern | Separater Service |
| Performance | Bis ~100K Zeilen problemlos | Unbegrenzt |
| Kosten | 0 | ~5-15 EUR/Monat (managed) |

**SQLite ist die richtige Wahl bis du >10 gleichzeitige Nutzer hast.**

## Migration-Pfad zu Postgres

Wenn du von Anfang an ein ORM/Query Builder nutzt, ist der Wechsel spaeter 1 Config-Aenderung:

```python
# SQLite (jetzt)
DATABASE_URL = "sqlite:///./invoices.db"

# Postgres (spaeter)
DATABASE_URL = "postgresql://user:pass@host:5432/invoices"
```

Mit SQLAlchemy oder Drizzle aendert sich am Code NICHTS.

## Schema (einfach halten)

```sql
CREATE TABLE invoices (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    lieferant TEXT NOT NULL,
    rechnungsnummer TEXT,
    datum DATE,
    waehrung TEXT DEFAULT 'EUR',
    netto_gesamt REAL,
    mwst_gesamt REAL,
    brutto_gesamt REAL,
    raw_json TEXT,          -- Das komplette Claude-Ergebnis
    pdf_filename TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE line_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    invoice_id INTEGER REFERENCES invoices(id),
    beschreibung TEXT,
    menge REAL,
    einheit TEXT,
    einzelpreis REAL,
    gesamtpreis REAL
);
```

**Wichtig:** `raw_json` speichert das komplette Claude-Ergebnis. Falls du spaeter neue Felder brauchst, kannst du sie aus dem gespeicherten JSON re-extrahieren ohne PDFs nochmal zu verarbeiten.
