# Architektonischer Grundriss

Der Zielaufbau fuer AI ERP / bitsbridge.net — inspiriert von OpenClaw, erweitert fuer Enterprise-Anforderungen:

```
┌────────────────────────────────────────────────┐
│  Channels: Web-UI, Messenger, Email, API       │
│  + EDI, IoT-Sensoren, E-Invoicing              │
├────────────────────────────────────────────────┤
│  Gateway (Auth, Routing, Multi-Tenancy)        │
│  WebSocket + Event-Bus + Rate Limiting          │
├────────────────────────────────────────────────┤
│  Agent Layer (viele spezialisierte Agenten)    │
│  - Procurement Agent                            │
│  - Finance Agent                                │
│  - HR Agent                                     │
│  - Compliance Agent                             │
│  - ... kommunizieren untereinander              │
├────────────────────────────────────────────────┤
│  Skill Registry (MCP-basiert, Markdown)        │
│  - Branchen-Templates (Handwerk, E-Commerce)   │
│  - Prozess-Bausteine (Mahnwesen, Onboarding)  │
│  - Kunden-eigene Skills                         │
├────────────────────────────────────────────────┤
│  Deterministic Core Engine                     │
│  - Ledger, Rechnungen, Bestaende               │
│  - Audit-Log, Unveraenderbarkeit               │
│  - Workflow-Engine (Freigaben, 4-Augen)        │
├────────────────────────────────────────────────┤
│  Data Layer (Postgres + Vector Store)          │
│  - ACID fuer Transaktionen                      │
│  - Vector DB fuer semantische Suche            │
│  - JSONL fuer Agent-Memory                      │
└────────────────────────────────────────────────┘
```

## Die Schluesselentscheidung: Agentische Huelle vs. Deterministischer Kern

| Aspekt | Deterministischer Kern | Agentische Huelle |
|--------|----------------------|-------------------|
| **Was** | Buchungen, Zahlungen, Steuern | Kommunikation, Analyse, Vorbereitung |
| **Wie** | Validierte Business-Logik | LLM-gesteuerte Agenten |
| **Fehlertoleranz** | Null (muss korrekt sein) | Hoch (kann nachbessern) |
| **Audit** | Unveraenderbar, signiert | Nachvollziehbar, korrigierbar |
| **Beispiel** | Buchungssatz erzeugen | "Schlag mir eine Kostenstelle vor" |

## Naechster Schritt
Die architektonische Schluesselentscheidung — wo genau verlaeuft die Grenze zwischen deterministischem Kern und agentischer Huelle? — muss fuer jedes Modul einzeln getroffen werden.
