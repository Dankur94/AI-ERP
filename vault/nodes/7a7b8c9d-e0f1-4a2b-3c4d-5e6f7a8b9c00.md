# Auth-Strategie — Magic Link

## Warum Magic Link die beste MVP-Auth ist

```
Nutzer gibt Email ein
  → Server generiert Token (UUID + Ablauf 15min)
  → Email mit Link: app.example.com/auth/verify?token=abc123
  → Nutzer klickt Link
  → Server prueft Token, setzt Session-Cookie
  → Eingeloggt.
```

## Implementierung (~60 Zeilen)

```python
import uuid, secrets
from datetime import datetime, timedelta

# In-Memory Token Store (fuer MVP reicht das)
pending_tokens = {}

def create_magic_link(email: str) -> str:
    token = secrets.token_urlsafe(32)
    pending_tokens[token] = {
        "email": email,
        "expires": datetime.now() + timedelta(minutes=15)
    }
    return f"https://app.example.com/auth/verify?token={token}"

def verify_token(token: str) -> str | None:
    data = pending_tokens.pop(token, None)
    if data and data["expires"] > datetime.now():
        return data["email"]
    return None
```

Fuer MVP: **Resend** (kostenlos bis 100 Emails/Tag, 1 API-Call):

## Session

Nach Verifizierung: Signed Cookie mit Email setzen.
Bei jedem Request pruefen ob Cookie da und gueltig ist.

**Kein:** JWT (Overkill), kein OAuth (zu komplex), kein Passwort (unsicherer).

## Multi-Tenancy (minimal)

Jede Query bekommt ein `WHERE user_email = :email`. Das ist alles.
Kein `tenant_id`, keine Organizations, keine Teams. Kommt spaeter wenn noetig.
