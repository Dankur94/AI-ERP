# Shopify — Architektur-Lessons

| Shopify-Prinzip | Bedeutung fuer AI ERP |
|---|---|
| Eigenes Problem zuerst loesen | Luetke baute Software fuer seinen Snowboard-Shop, nicht eine "Plattform" — das Produkt entstand aus echtem Bedarf |
| Monolith + Boundary-Tooling | 2.8M Zeilen Rails-Monolith, skaliert mit Packwerk (statische Grenz-Analyse), nicht Microservices |
| Sharding-Key = Geschaeftsmodell | `shop_id` als universeller Key — Shops sind natuerlich isoliert, Multi-Tenancy faellt gratis raus |
| Sandboxing fuer Drittanbieter | Fremder Code laeuft in Sandboxes (WebWorker fuer UI, WebAssembly fuer Logik) — Platform ohne Risiko |
| Enterprise = Feature-Flag, kein Fork | Shopify Plus ist derselbe Code wie Basic, Features per Flag freigeschaltet |
| Revenue trackt Kundenerfolg | 73% Umsatz aus Transaktionsgebuehren, nicht Abo — Shopify verdient wenn der Haendler verdient |

**Kontrast:** Odoo = viele Module. Stripe = ein Monolith. Shopify = Monolith mit erzwungenen Grenzen (Packwerk). Alle drei: **geteiltes Datenmodell im Kern**.

**Referenz:** Snowboard-Shop (2004) → SaaS Launch (2006) → App Store (2009) → Plus/Enterprise (2014) → $11.5B Revenue, $292B GMV (2025)
