# Phase 1 — Requirements Engineering (2 Wochen)

**Woche:** 2-3
**Status:** Offen

## Was

Klaeren, *was* du baust — mit 6 Techniken.

## Warum

Ohne klare Anforderungen baust du das Falsche.

## Techniken

1. **Vision Statement** — Ein Satz, der das Produkt beschreibt → siehe 5.2.1
2. **Personas** — 5 konkrete Personas aus verschiedenen Branchen → siehe 5.2.2
3. **Jobs to Be Done (JTBD)** — Was will der Nutzer *wirklich* erreichen?
4. **Journey Maps** — Schritt-fuer-Schritt durch die Nutzererfahrung
5. **User Stories** — "Als [Rolle] will ich [Aktion], damit [Nutzen]"
6. **MoSCoW-Priorisierung** — Must/Should/Could/Won't fuer MVP-Scope

## Output / Deliverables

- Vision Statement
- 5 Personas (Trading, Fashion, F&B, Restaurant, Healthcare)
- Journey Maps fuer Haupt-Workflows
- 30-50 User Stories
- MVP-Definition (was ist drin, was nicht)

## Checkliste

- [ ] Vision Statement formuliert
- [ ] Personas erstellt (5 Stueck)
- [ ] JTBD-Analyse durchgefuehrt
- [ ] Journey Maps fuer min. 2 Workflows
- [ ] User Stories geschrieben und priorisiert
- [ ] MoSCoW-Priorisierung abgeschlossen
- [ ] MVP-Scope klar definiert
