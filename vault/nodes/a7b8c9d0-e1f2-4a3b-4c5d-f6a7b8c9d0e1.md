# Channel Layer & Gateway

## Channel Layer (Ebene 1 — Eingang)
Messaging-Integrationen mit ueber 50 Kanaelen, jeder mit eigenem Adapter:

| Kategorie | Kanaele |
|-----------|---------|
| Messenger | WhatsApp, Telegram, Signal, iMessage |
| Business  | Slack, Discord, Microsoft Teams, Google Chat |
| Open      | Matrix, IRC |
| Eigene    | macOS-App, Web-UI, CLI |

### Adapter-Pattern
Jeder Kanal implementiert ein gemeinsames Interface:
- `receiveMessage()` — Nachricht empfangen und normalisieren
- `sendMessage()` — Antwort im kanalspezifischen Format senden
- `handleMedia()` — Bilder, Dateien, Voice-Messages verarbeiten

## Gateway (Ebene 2 — Hub)
Das zentrale Nervensystem als WebSocket-Server:
- **Session Management**: Zustand aller aktiven Konversationen
- **Routing**: Nachricht → richtiger Agent/Skill
- **Authentifizierung**: Verbindungen verifizieren
- **Event-Handling**: Lifecycle-Events verarbeiten

### Fuer AI ERP
Der Channel Layer wuerde um **geschaeftsspezifische Eingaenge** erweitert:
- EDI-Schnittstellen (Lieferanten-Kommunikation)
- E-Mail-Eingang (Rechnungen, Bestellungen)
- API-Endpoints (B2B-Integration)
- IoT-Sensoren (Wareneingang, Produktion)
