# Modul 5 — Warnungen & Alerts

**Was:** Regeln definieren, System ueberwacht automatisch, benachrichtigt bei Abweichung.

**Beispiele:** Dead-Stock-Alert, Nachbestell-Warnung, No-Show-Erinnerung.

## Scope

- [x] Ueberfaellige Zahlungen erkennen (payment_due < heute, status != confirmed)
- [x] In-App-Anzeige mit Badge im Header
- [x] Alert-Detailseite mit "Days Overdue"
- [ ] Regel-Engine (Schwellenwert → Aktion) — auf Kundennachfrage
- [ ] Benachrichtigung per E-Mail — auf Kundennachfrage
- [ ] Benachrichtigung per WhatsApp — spaeter (Master-MD)
- [ ] Alert-Historie

## Status

Overdue-Alert gebaut (2026-04-25) als Beispiel. Weitere Alerts auf Kundennachfrage (L7).

## Technische Details

- Backend: GET /api/alerts — berechnet, kein neues Datenmodell
- Overdue: payment_due < heute AND status != confirmed
- Frontend: Badge im Header (pollt alle 60s), /alerts Route
- L3: Tenant-isoliert
- Kein E-Mail, kein Queue, kein Rule-Engine (L4, L11)

## Spaeter (auf Nachfrage)

- Regel-Engine: Feld + Operator + Schwellenwert + Aktion
- E-Mail-Benachrichtigungen (SMTP)
- "Rechnung seit X Tagen nicht bestaetigt"
- "Grosse Rechnung" Alert (total > Schwellenwert)
- Alert-Historie (was wurde wann ausgeloest)
