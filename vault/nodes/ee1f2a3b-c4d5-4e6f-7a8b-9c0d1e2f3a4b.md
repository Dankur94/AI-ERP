# Phase 9 — Pilotkunde onboarden und iterieren (laufend)

**Woche:** ab 18
**Status:** Offen

## Was

Ersten zahlenden Kunden live nehmen, Feedback sammeln, anpassen.

## Warum

Echtes Nutzerfeedback ist wertvoller als alle Annahmen zusammen.

## Output / Deliverables

- Kundenreferenz
- Case Study
- Erste Iteration basierend auf echten Schmerzen

## Ablauf

1. **Pilotkunde identifizieren** — Idealerweise bekannter Kontakt, mittelgrosser Importeur
2. **Onboarding** — Gemeinsam Daten einpflegen, System erklaeren
3. **Begleitetes Quartal** — Erste CBAM-Deklaration gemeinsam durchfuehren
4. **Feedback-Schleifen** — Woechentliche Check-ins, Pain-Points sammeln
5. **Iterieren** — Schnelle Fixes, Feature-Anpassungen basierend auf Feedback
6. **Case Study** — Ergebnisse dokumentieren fuer Marketing

## Checkliste

- [ ] Pilotkunde gefunden und Vertrag/Vereinbarung
- [ ] Onboarding durchgefuehrt
- [ ] Erste Daten im System
- [ ] Erste Deklaration gemeinsam erstellt
- [ ] Feedback gesammelt und priorisiert
- [ ] Top-3 Feedback-Punkte umgesetzt
