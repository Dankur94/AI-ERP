# Woche 3 — Ergebnisse in DB

## Ziel

Jede verarbeitete Rechnung wird gespeichert und ist wiederfindbar.

## Konkret

```
Browser:
  [Upload wie bisher]

  → Verarbeitung...

  → Gespeichert! (Rechnungs-ID: #42)

  [Alle Rechnungen anzeigen]

  → Tabelle:
    #42 | Shenzhen Electronics | INV-2026-0042 | 4,850.00 USD | 15.04.2026
    #41 | Taiwan Chips Ltd.    | INV-2026-0038 | 12,300.00 EUR | 12.04.2026
    ...

  [Suche: "Shenzhen"] → filtert live
```

## Warum jetzt DB?

Ohne Persistenz ist das Tool ein Wegwerf-Script. Mit DB wird es ein **System**:
- Kunde kann zurueckschauen ("Was hat Lieferant X letzten Monat berechnet?")
- Du hast Daten fuer spaetere Features (Dashboards, Exporte, Anomalie-Erkennung)
- Audit-Trail entsteht automatisch

## Was man braucht

1. SQLite (fuer MVP perfekt — kein Server, eine Datei, null Config)
2. Ein ORM oder Query Builder (Drizzle, Prisma, oder SQLAlchemy)
3. Migrations-Setup (auch bei SQLite — spaeterer Wechsel zu Postgres wird trivial)
4. Eine Listenansicht mit Suchfeld

## Kein:
- Kein Postgres (kommt erst mit mehreren Nutzern)
- Keine komplexen Relationen
- Keine Pagination (erst ab >100 Eintraege relevant)
- Kein Export (kommt Woche 5+)
