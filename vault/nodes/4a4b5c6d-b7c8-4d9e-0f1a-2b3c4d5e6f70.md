# Stack-Entscheidung — FastAPI vs Express

## Empfehlung: FastAPI (Python)

| Kriterium | FastAPI (Python) | Express (Node.js) |
|---|---|---|
| Claude SDK | `anthropic` (Python-first) | `@anthropic-ai/sdk` (gut) |
| PDF-Verarbeitung | `pdf2image`, `PyMuPDF` — ausgereift | `pdf-parse` — weniger robust |
| Lernkurve | Einfach wenn Python bekannt | Einfach wenn JS bekannt |
| Typed Responses | Pydantic (JSON-Validierung gratis) | Zod (extra Setup) |
| Deployment | uvicorn, 1 Befehl | node, 1 Befehl |
| Async | Native | Native |

## Warum Python gewinnt fuer diesen Use Case

1. **Anthropic SDK ist Python-first** — beste Docs, meiste Beispiele
2. **Pydantic** — definiere dein JSON-Schema als Python-Klasse, Validierung automatisch
3. **Data-Ecosystem** — falls spaeter Pandas/Analytics dazukommt, kein Sprachwechsel

## Minimal-Setup

```
projekt/
  main.py          # FastAPI App (~50 Zeilen)
  extract.py       # Claude-Aufruf (~30 Zeilen)
  templates/
    index.html     # Upload-Formular
    result.html    # Ergebnis-Anzeige
  requirements.txt # anthropic, fastapi, uvicorn, python-multipart
```

**Kein:** Framework-Overkill (Django), kein Frontend-Framework (React), kein Docker (noch nicht).

## Alternative: Wenn du JS bevorzugst

Dann Hono oder Express + Zod. Funktioniert auch, aber PDF-Libraries sind schwaecher.
Die Entscheidung ist nicht kritisch — wichtiger ist dass du ANFAENGST.
