# Prompt Engineering fuer PDF-Extraktion

## Der wichtigste Teil des ganzen Produkts

Das Prompt ist dein Algorithmus. Die Qualitaet der Extraktion steht und faellt damit.

## Basis-Prompt (funktioniert sofort)

```
Du bist ein Rechnungs-Parser. Extrahiere ALLE Daten aus dieser Rechnung
als JSON. Halte dich EXAKT an dieses Schema:

{
  "lieferant": "Name des Lieferanten",
  "lieferant_adresse": "Vollstaendige Adresse",
  "rechnungsnummer": "Exakte Rechnungsnummer",
  "datum": "YYYY-MM-DD",
  "faellig_am": "YYYY-MM-DD oder null",
  "waehrung": "ISO 4217 Code (EUR, USD, etc.)",
  "netto_gesamt": 0.00,
  "mwst_gesamt": 0.00,
  "brutto_gesamt": 0.00,
  "positionen": [
    {
      "beschreibung": "Artikelbezeichnung",
      "menge": 0,
      "einheit": "Stueck/kg/etc.",
      "einzelpreis": 0.00,
      "gesamtpreis": 0.00
    }
  ]
}

Regeln:
- Gib NUR valides JSON zurueck, keinen anderen Text
- Wenn ein Feld nicht vorhanden ist, setze null
- Betraege immer als Zahl, nie als String
- Datum immer im Format YYYY-MM-DD
```

## Prompt-Verbesserungen (iterativ einfuehren)

| Iteration | Verbesserung | Wann |
|---|---|---|
| v1 | Basis-Prompt oben | Sofort |
| v2 | + Konfidenz-Score pro Feld (0-1) | Wenn du merkst dass manche Felder unsicher sind |
| v3 | + Branchen-spezifische Felder | Wenn Kunden spezifische Anforderungen haben |
| v4 | + Few-Shot Examples | Wenn bestimmte Rechnungsformate Probleme machen |

## Typische Fehler und Fixes

| Problem | Ursache | Fix |
|---|---|---|
| Betraege als String "1.234,56" | Deutsche Zahlenformate | Explizit sagen: "Dezimaltrennzeichen ist Punkt" |
| MwSt fehlt | Nicht auf Rechnung oder uebersehen | null statt 0 zurueckgeben lassen |
| Falsche Waehrung | Symbol mehrdeutig ($ = USD/CAD/AUD) | "Leite Waehrung aus Land des Lieferanten ab" |
| Positionen fehlen | Tabelle schlecht erkannt | PDF als Bild senden statt direkt |

## Golden Rule

**Teste mit 10 echten Rechnungen verschiedener Lieferanten.** Nicht 1, nicht 3 — zehn.
Jede Rechnung die Probleme macht zeigt dir eine Luecke im Prompt.
