# Nach Woche 4 — Iterate oder Pivot

## Du hast ein deployed Produkt. Was jetzt?

### Szenario A: Kunden zahlen → ITERATE

Die naechsten Features nach Prioritaet:

| Prio | Feature | Warum |
|---|---|---|
| 1 | CSV/Excel Export | Kunden wollen Daten in ihre Systeme importieren |
| 2 | Bulk Upload (10 PDFs auf einmal) | Zeitersparnis skalieren |
| 3 | Dashboard mit Ausgaben-Uebersicht | Vom Tool zum System |
| 4 | Automatische Kategorisierung | "Buero", "Logistik", "Material" |
| 5 | API-Zugang | Fuer technische Kunden die integrieren wollen |
| 6 | DATEV-Export | Steuerberater-Kompatibilitaet (Deutschland-Killer-Feature) |

**Regel:** Nur Features bauen die Kunden aktiv anfragen. Nicht raten.

### Szenario B: Niemand zahlt → PIVOT

Nicht aufgeben — umlenken. Moegliche Pivots:

| Pivot | Wie |
|---|---|
| Anderer Dokumenttyp | Lieferscheine, Zollpapiere, Vertraege statt Rechnungen |
| Andere Zielgruppe | Steuerberater statt Firmen direkt |
| Anderes Geschaeftsmodell | Per-Rechnung statt Abo (Pay-as-you-go) |
| White-Label | Anderen SaaS-Tools die Extraktion als API anbieten |

### Szenario C: Interesse aber kein Kauf → DIAGNOSE

Frage die Leute die sich angemeldet aber nicht bezahlt haben:
- "Was hat Sie davon abgehalten?" (Preis? Feature? Vertrauen?)
- "Was muesste das Tool koennen damit Sie es nutzen?"
- "Wie loesen Sie das Problem aktuell?"

## Die eine Metrik die zaehlt

**MRR (Monthly Recurring Revenue)**

- Woche 4: Ziel 1 zahlender Kunde (49 EUR MRR)
- Monat 3: Ziel 10 Kunden (490 EUR MRR)
- Monat 6: Ziel 50 Kunden (2.450 EUR MRR)
- Monat 12: Ziel 200 Kunden (9.800 EUR MRR)

Alles andere (Features, Code-Qualitaet, Tech-Stack) ist der MRR untergeordnet.

## Die Falle

Die Falle nach Woche 4 ist die gleiche wie vorher:
**Statt mit Kunden zu reden, ein neues Feature bauen.**

Features bauen fuehlt sich produktiv an.
Kunden fragen fuehlt sich unangenehm an.
Deshalb machen alle das Falsche.
