# Tool/Skill Layer & Output

## Ebene 4 — Tool/Skill Layer (Faehigkeiten)

### Built-in Tools
| Tool | Funktion |
|------|----------|
| Browser | Webseiten navigieren, Formulare ausfuellen |
| Dateisystem | Lesen, Schreiben, Organisieren |
| Shell | Beliebige Kommandos ausfuehren |
| Cron | Zeitgesteuerte Aufgaben |
| Webhooks | Externe Events empfangen |
| Kamera/Screen | Hardware-Zugriff |

### Erweiterbar via
- **Skills**: Markdown-basierte Instruktionen mit Skripten (siehe Skills-Kapitel)
- **MCP-Server**: Standardisierte Tool-Bereitstellung (siehe MCP-Kapitel)
- **Custom Tools**: Eigene Tool-Implementierungen

## Ebene 5 — Output Layer (Aktion)

Zwei Ausfuehrungsmodi:

### Direkt auf dem Host
- Schnell, kein Overhead
- Geeignet fuer vertrauenswuerdige Operationen
- Risiko: fehlerhafter Code kann Host beschaedigen

### Docker-Container (Sandbox)
- Isolierte Ausfuehrung
- Sicher fuer unbekannten/generierten Code
- **Kritisch fuer Kundendaten** — verhindert Seiteneffekte
- Leichter Overhead durch Container-Start

## Fuer AI ERP
- **Deterministischer Kern** (Buchhaltung, Zahlungen) laeuft NICHT agentisch, sondern in validierter Business-Logik
- **Agentische Huelle** (Kommunikation, Analyse, Vorbereitung) nutzt Tool/Skill-Layer
- Docker-Sandboxing ist Pflicht fuer alle kundendatenverarbeitenden Operationen
