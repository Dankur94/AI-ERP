# Implikationen fuer AI ERP

OpenClaws Architektur liefert eine Blaupause — aber an drei fundamentalen Stellen muss anders gedacht werden, um ein ERP daraus zu machen.

## Die drei Paradigmenwechsel

### 1. Kanal-First → Prozess-First
**OpenClaw**: Messenger rein, Aktion raus (kanal-zentriert)
**AI ERP**: Geschaeftsereignisse treiben Prozesse (Wareneingang, Zahlungseingang, Bestellung)
→ Braucht eine **Event-Engine**, die OpenClaws Philosophie erweitert

### 2. Single-User → Multi-User
**OpenClaw**: Persoenlicher Assistent fuer eine Person
**AI ERP**: Viele Nutzer, Rollen, Berechtigungen, gleichzeitige Zugriffe, Mandantentrennung
→ Hub-and-Spoke funktioniert, aber der Hub muss **Nutzer, Rollen und Datenisolation** sauber trennen

### 3. Improvisation → Determinismus (wo noetig)
**OpenClaw**: Agenten improvisieren — das ist ihre Staerke
**AI ERP**: Buchhaltung, Zahlungsabwicklung, Steuermeldung duerfen **nicht** improvisieren
→ **Zweischichtiger Ansatz**:
  - Deterministische Kern-Engine fuer bilanzrelevante Transaktionen (unveraenderlich, auditierbar)
  - Agentische Huelle fuer alles drumherum (Kommunikation, Entscheidungsvorbereitung, Automatisierung)
