# MCP als Standard-Protokoll

## Was ist MCP?
**Model Context Protocol** — ein von Anthropic initiierter offener Standard, der definiert, wie AI-Modelle mit Tools interagieren. Statt proprietaerer Tool-Definitionen ein universelles Protokoll.

## Wie OpenClaw MCP nutzt

### Als MCP-Server-Betreiber
OpenClaw betreibt MCP-Server fuer seine eingebauten Capabilities:
- Browser-Tool als MCP-Service
- Dateisystem-Tool als MCP-Service
- Shell-Tool als MCP-Service

### Als MCP-Client
OpenClaw verbindet sich mit **externen MCP-Servern** fuer zusaetzliche Funktionalitaet:
- Datenbank-Zugriff via MCP
- API-Integrationen via MCP
- Spezialisierte Tools von Drittanbietern

## Strategische Bedeutung
- **Kein Lock-in**: Tools fuer OpenClaw funktionieren auch mit Claude, ChatGPT und anderen MCP-kompatiblen Systemen
- **Oekosystem**: Wachsendes Oekosystem von MCP-Servern, die man einfach anbinden kann
- **Standardisierung**: Eine Tool-Definition, viele Konsumenten

## Fuer AI ERP
MCP ist der **Integrations-Standard** fuer AI ERP:
- **DATEV-MCP-Server**: Buchhaltungsdaten standardisiert bereitstellen
- **Banking-MCP-Server**: Kontobewegungen, Ueberweisungen
- **ELSTER-MCP-Server**: Steuermeldungen automatisieren
- **EDI-MCP-Server**: Lieferanten-Kommunikation
- **Eigene MCP-Server**: Jedes AI-ERP-Modul exponiert seine Faehigkeiten als MCP-Service

Vorteil: Kunden koennen AI-ERP-Tools auch mit anderen AI-Systemen nutzen — kein Vendor-Lock-in.
